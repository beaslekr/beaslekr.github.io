Setup

This circuit adds together two binary numbers, each with two digits.
It takes four input switches, representing the digits that comprise each number.
The three lights represent each digit of the output sum.

Finding 1s bit

To receive the output’s 1s digit, we take XOR of the 1s digits of the inputs.
If we look solely at the input 1s digits, this is because 0 + 1 and 1 + 0 both yield 0, but 1 + 1 yields 10 (or, 0 with a 1 carried to the 10s digit) and 0 + 0 yields 0.

Finding 10s bit

To receive the output’s 10s digit, the number of 1s found in the input 10s digits and the carried 10s digit must be odd.
Thus, we take the carried digit (represented by the AND of the two 1s digits) and XOR it together with the XOR of the two 10s digits (though order of the XOR’s does not impact the output).
XOR only returns 1 if its inputs contain an odd number of 1s, so applying it twice to three inputs returns our desired output.

Finding 100s bit

To receive the output’s 100s digit, at least two 10s digits (including the carry digit) need to be a 1.
This is equivalent to either both input 10s digits being 1 OR one input 10s digit being a 1 and the carry digit also being 1.
Our diagram reflects this.

Setup

Like the addition circuit, this circuit’s switches and lights are arranged in the same fashion, apart from a resultant 100s bit (which would always have a value of 0).

Finding 10s bit

The resulting 10s bit can be represented as c - d - (the borrow bit), where c and d are the input 10s bits.
In other words, in order for e to be 1, c must be 1 AND d must be 0 AND there cannot be any borrowing.
This is represented by the upper right of the circuit.

Must-borrow Bit

The borrow bit is only 1 if the second number’s 1s bit (b) must borrow from its 10s bit (d).
This is only true if a is 0 and b is 1, as shown.

Can-borrow Bit

The 1s place can only borrow from the 10s place if there is an extra 10 to be borrowed from.
This is only the case when c is 1 and d is 0, as shown.
The resulting AND gate is the “can be borrowed from” bit.

Final Result

The resulting 1s bit is more complicated.
If borrowing is necessary, AND we can borrow, then we are performing the operation 10 - 01 = 01, which has 1s value 1.
If borrowing is necessary, but we cannot borrow, then we do not borrow and simply receive a value of 0.
If we do not need to borrow, then we can only achieve a 1s bit of 1 if the problem is of form c1 - d0 = ef.
HOWEVER, this is only the case if c - d is not 0 - 1 (marked by the “note^” bit).
If that is the case, our resulting value is necessarily 00 and thus the 1s bit is 0.
Otherwise, we receive a 1s value of 1.
